## From the RM-06 independent security review (2026-07-07)
The review's MUST-FIX (#1 DEFINER search_path + revoke CREATE; #2 consent-gate all raw reads) and SHOULD-FIX (#3 unforgeable audit; #4 no consent oracle; #5 FORCE RLS; #6 assignments provenance) were fixed in migration `0006_hardening.sql`. The following were consciously deferred:

- **SEC-REV-07 — Legacy-PIN name lockout is a targeted DoS.** `record_attempts` (0001) locks a known player name for 15 min after 5 bad PINs, repeatable. *Deferred because* name+PIN auth is being retired for real Google OAuth (SPEC RM-12) and the legacy game rows are frozen (RM-15). Revisit only if PIN auth outlives the cutover; then IP-scope the throttle / degrade to CAPTCHA. Severity: low.
- **SEC-REV-08 — Direct `child_context_pack` / `authorize` callers bypass the gateway's verify/moderate/audit.** An authenticated user can call these RPCs directly, skipping the Edge Function's choke points. *Deferred because* the pack is a name-free whitelist of data the caller is already authorized to read (same as `mastery_select`), and verify/moderate only guard generated text the RPC never produces; the only loss is audit-completeness for direct reads (never audited anyway). If complete AI-access auditing becomes required, route pack access behind one audited RPC and revoke direct `execute` of `child_context_pack` from `authenticated`. Severity: low. (Relates to KER-1/KER-6.)
- **SEC-REV-09 — `teaching_artifacts` INSERT does not constrain `visibility_scope`.** A writer may set any scope on their own artifact for a child they write to. *Deferred because* no cross-family leak results — `teaching_artifacts_select` still gates every read by `can_view_child` + consent; this is authorization hygiene only. Constrain settable scope per `author_role` if scopes gain behavioral meaning. Severity: low. (Relates to DM-15.)
- **SEC-REV-10 — Edge Function CORS `Access-Control-Allow-Origin: *`.** *Cleared as acceptable* — the function is bearer-token (JWT) authorized, not cookie-based, so `*` does not enable credential replay. Recorded for completeness. Severity: informational. (Relates to SEC-01.)

- **AUDIT-HARDENING (stronger option, deferred).** `0006` made audit writes unforgeable via decision-recompute + detail cap + definer source stamp (finding #3, minimal). The larger restructure — revoke `write_audit` from `authenticated` and route the whole spine through one audited RPC so clients can only trigger the full path, never a bare audit write — remains available if evidentiary requirements tighten. (SEC-07 / ACC-06.)
