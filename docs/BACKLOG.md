The review's MUST-FIX (#1 DEFINER search_path + revoke CREATE; #2 consent-gate all raw reads) and SHOULD-FIX (#3 unforgeable audit; #4 no consent oracle; #5 FORCE RLS; #6 assignments provenance) were fixed in migration `0006_hardening.sql`. The following were consciously deferred:

Revisit only if PIN auth outlives the cutover; then IP-scope the throttle / degrade to CAPTCHA. Severity: low.
- **SEC-REV-08 ‚Äî Direct `child_context_pack` / `authorize` callers bypass the gateway's verify/moderate/audit.** An authenticated user can call these RPCs directly, skipping the Edge Function's choke points. *Deferred because* the pack is a name-free whitelist of data the caller is already authorized to read (same as `mastery_select`), and verify/moderate only guard generated text the RPC never produces; the only loss is audit-completeness for direct reads (never audited anyway). If complete AI-access auditing becomes required, route pack access behind one audited RPC and revoke direct `execute` of `child_context_pack` from `authenticated`. Severity: low. (Relates to KER-1/KER-6.)
- **SEC-REV-09 ‚Äî `teaching_artifacts` INSERT does not constrain `visibility_scope`.** A writer may set any scope on their own artifact for a child they write to. *Deferred because* no cross-family leak results ‚Äî `teaching_artifacts_select` still gates every read by `can_view_child` + consent; this is authorization hygiene only. Constrain settable scope per `author_role` if scopes gain behavioral meaning. Severity: low. (Relates to DM-15.)
- **SEC-REV-10 ‚Äî Edge Function CORS `Access-Control-Allow-Origin: *`.** *Cleared as acceptable* ‚Äî the function is bearer-token (JWT) authorized, not cookie-based, so `*` does not enable credential replay. Recorded for completeness. Severity: informational. (Relates to SEC-01.)

Revisit by first-principles review if a case needs adults and children in one id space (e.g. an adult who is also a "learner").
