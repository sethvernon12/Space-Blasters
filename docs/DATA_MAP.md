# DATA MAP — game event ↔ stage ↔ CCSS ↔ mastery schema

Everything below was read directly from `index.html` on 2026-07-03 (not assumed).
`db/scripts/validate-schema.mjs` re-parses `index.html` on every run and fails if this
map drifts from the code.

## 1. The answer event the game ACTUALLY emits today

Every answered/missed problem flows through `recordAnswer(entry)` → `game.log` and the
optional `window.onMathAnswer(evt)` callback:

| field | type | example | notes |
|---|---|---|---|
| `text` | string | `"4 − 2"` | problem as displayed; no PII |
| `correctAnswer` | int | `2` | |
| `chosen` | int \| null | `3` / `null` | `null` when the problem fell (missed) |
| `correct` | bool | `false` | |
| `missed` | bool (only present when true) | `true` | fall-through miss |
| `level` | int | `7` | global curriculum level |
| `skill` | string | `"subtraction"` | one of the 13 coarse tags below |
| `stage` | string | `"Subtract within 5"` | display name |
| `stageIndex` | int | `1` | index into the 23-stage `STAGES` ladder — **for review/preview problems this is the problem's stage, not the player's** |
| `time` | float (s) | `27.8` | **cumulative seconds since run start — NOT per-problem latency** |
| `mode` * | string | `"journey"` | `journey\|beginner\|intermediate\|advanced\|expert` |
| `pilot` * | string | `"Nova"` | display name — **never stored in `attempts` (PII rule)** |

\* added only on the `onMathAnswer` callback, not in `game.log`.

**Attribution rule:** an event maps to a mastery skill via
`skill_id = STAGES[evt.stageIndex].key` (= `skills.position` in the DB). The coarse
`skill` tag is informational (`skills.category`) — one tag spans several stages, and
`mixMD` emits two different tags.

## 4. Signals the mastery model wants that the game does NOT emit yet

All exist as **nullable, unpopulated** columns (see comments in
`supabase/migrations/0001_mastery.sql`):

| signal | column(s) | will come from |
|---|---|---|
| per-problem latency | `attempts.latency_ms`, `mastery.fluency_latency_ms_median`, `fluency_trend` | Phase-1 follow-up: game emits `latencyMs` per problem (today only cumulative `time`) |
| misconception / slip / guess classification | `attempts.result` extra enum values, `attempts.misconception_id`, `child_skill_misconception.*` | Phase 2: misconception-tagged distractors + response-pattern classifier |
| spaced retention | `mastery.retention_last_success_at`, `retention_strength` | Phase 2: review scheduler (~1d, 3d, 1wk, 3wk, monthly) |
| confidence / metacognition | `mastery.confidence` | Phase 6/7: self-prediction prompts |
| transfer (handwritten / open-ended) | `mastery.transfer_success_count`, `transfer_last_success_at` | Phase 5: photo-grading pipeline |
